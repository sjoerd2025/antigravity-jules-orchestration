# Implement New Feature

Implement a new feature using parallel agent workflow.

## Usage

```
/implement-feature [feature description]
```

## Parallel Agent Tasks

### Agent 1: Planning
Create implementation plan:
- Analyze feature requirements
- Identify affected files
- Design API changes if needed
- Plan test cases

### Agent 2: Research Existing Code
Research relevant existing code:
- Find related functionality
- Understand current patterns
- Identify reusable components
- Check for conflicts

### Agent 3: Design API
If API changes needed:
- Design new endpoints
- Define request/response schemas
- Plan authentication requirements
- Design error handling

### Agent 4: Design Tests
Plan test coverage:
- Unit test cases
- Integration test cases
- API test scenarios
- Edge cases to cover

## Implementation Steps

1. Review parallel agent findings
2. Create feature branch
3. Implement core functionality
4. Add error handling
5. Add tests
6. Update documentation
7. Create PR

## Output

Provide implementation plan with:
- Files to modify
- New files to create
- API changes
- Test cases
- Documentation updates
